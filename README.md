# Research Viewer

Interactive research canvas for the oxFlow corpus. Two workspaces in one monorepo:

- **`studio/`** — Next.js 16 web app (Liveblocks-backed collaborative canvas, concept-tree viewer, markdown reader). Deployed to Vercel at [oxflow-studio.vercel.app](https://oxflow-studio.vercel.app).
- **`studio-bridge/`** — tiny local WebSocket server that spawns your own `claude` CLI for chat + concept-tree generation. Runs on your machine; no cloud AI bill.

Nothing AI-related touches Vercel. Trees and chats are generated by the Claude CLI on the contributor's laptop.

---

## What you can do

- **Drop an `.md` on the canvas** (or paste markdown with ⌘V) → expands into a concept tree of individual, draggable React Flow nodes, one per concept, connected by edges.
- **Click Chat on any concept** → spawns a chat bubble pre-seeded with that branch's context so you can query it with Claude.
- **Drag a doc from the drawer** onto the canvas → placed as a DocCard for reference.
- **Double-click the canvas** → a new chat bubble at that point.
- **Rename a canvas** → click the title in the top-left, type a new name.
- **Invite teammates** — everything is Liveblocks-synced; cursors and edits appear live.

## Setup (first time)

### Prerequisites
- **Node.js ≥ 20** (Next.js 16 needs it).
- **Claude Code CLI** installed and authenticated (for the bridge). Install with `npm install -g @anthropic-ai/claude-code` then run `claude` once to sign in.
- A **Liveblocks** public + secret key. Ask a maintainer or create your own team at [liveblocks.io](https://liveblocks.io/).

### Clone + install

```bash
git clone https://github.com/361-coders-nz/Research-Viewer.git
cd Research-Viewer
```

### Studio (the web app)

```bash
cd studio
npm install
cp .env.example .env.local        # then fill in the Liveblocks keys
npm run dev                       # http://localhost:3000 (or :3001)
```

Required env vars (see `studio/.env.example`):
- `LIVEBLOCKS_SECRET_KEY` — server-side auth, fallback.
- `NEXT_PUBLIC_LIVEBLOCKS_PUBLIC_KEY` — direct browser connection, preferred.
- `NEXT_PUBLIC_DEFAULT_WORKSPACE` — the workspace room id everyone shares. E.g. `oxflow-team`.
- `NEXT_PUBLIC_BRIDGE_URL` — optional; defaults to `ws://localhost:4456`.

### Studio-bridge (chat + tree generation)

Needed whenever you want to use chat bubbles or drop markdown → tree. Runs locally only.

```bash
cd ../studio-bridge
npm install
npm start                         # → ws://127.0.0.1:4456
```

The top-bar **Bridge** badge in the studio turns green when connected. Keep the terminal open while you work.

## Deploy

```bash
cd studio
vercel --prod --yes
```

The Vercel project is `oxflow-studio` (aliased to `oxflow-studio.vercel.app`). Env vars live on Vercel — if one goes missing, `vercel env add NAME production`.

## Typical troubleshooting

- **"Bridge offline" toast** → `npm start` in `studio-bridge/`. The badge in the top-right of the studio tells you whether the socket is live.
- **Tree didn't appear after dropping an `.md`** → open the browser console, look for `[canvas-drop]` logs. A `tree.error` line means Claude's output didn't match the excerpt-verbatim rule; try a shorter or more explicit source.
- **Docs vanish after refresh** → workspace room mismatch. Make sure `NEXT_PUBLIC_DEFAULT_WORKSPACE` is the same across all machines.
- **Liveblocks auth errors** → verify both keys in `.env.local` are from the same Liveblocks project.
